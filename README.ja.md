# dejavu（デジャブ）

dejavu は Claude Code に長期記憶を追加するツールです。セッションを越えて生き残る記憶を与えてくれます。

**あなたが dejavu のコマンドを打つ場面はほとんどありません。** 最初の一度きりの設定を終えれば、
あとは Claude が自分で記憶したり思い出したりします。あなたは普段どおり話しかけるだけです。

## 何を解決するのか

たとえば何時間もかけて、あるプログラムをClaude Code で読み解いたとします。Claude はファイルを開き、
呼び出し関係を追い、なぜあの関数だけ妙な書き方をしているのかを突き止めました。作業は進みました。
セッションを閉じます。

翌朝、新しいセッションを開くと、**Claude は何も覚えていません。** 同じファイルを開き、同じ関数を追い、
昨日たどり着いたのとまったく同じ結論に、同じトークン数を使ってもう一度たどり着きます。

同じことは長いセッションの**途中でも**起きます。コンテキストが膨らむと応答の精度が落ちるので、
新しいセッションに切り替えたい。でも切り替えると、Claude が学んだことは全部消えます。

そこでdejavuの登場です。  
dejavu は、Claude が学んだことを自分で書き込み、必要なときに読み出すためのローカルなデータベースです。
セッションより長生きする記憶です。

```
セッション1                    セッション2（翌日）
──────────                    ────────────────────
Claudeがコードを読む           あなた:「昨日の続きしたい」
Claudeが理解する               Claudeが `dejavu resume` を実行
Claudeが分かったことを自動保存    Claudeが昨日の引き継ぎメモを読む
                               Claudeが止めた場所からそのまま再開
        │                                    ▲
        └────────── .dejavu/ ────────────────┘
                （ローカルのデータベースファイル）
```

dejavuの記憶は全てあなたのPC内で完結します。ネットワークアクセスも、アカウントも、同期サービスもありません。もちろん無料です。

## インストール（コマンドラインで１度だけ）

```bash
brew install alohayos/tap/dejavu
dejavu init --global
```

2行目は dejavu の指示書を `~/.claude/` に配置します。これが Claude Code に dejavu の使い方を教える部分です。一度やれば、あなたのマシンのすべてのプロジェクトで dejavu が使えるようになります。

> `dejavu` の代わりに、短いエイリアス `deja` でも動きます。このガイドでは `dejavu` を使って説明していきます。

新しいバージョンのdejavuに更新するときは `brew upgrade dejavu`。

## プロジェクトに導入する

リポジトリのルートで実行します。新規でも既存でも、言語は問いません。

```bash
dejavu init
```

**あなたが打つ必要のあるコマンドは、これが最後です。** 以下が作られます。

```
your-project/
├── .dejavu/                    ← 新規作成
│   ├── knowledge.db            （データベース。git 管理外）
│   ├── dejavu-triggers.md      （Claude 向けの指示書。git で追跡）
│   └── config.toml
├── CLAUDE.md                   ← 指示書を読み込む1行を追記
└── .gitignore                  ← データベースを除外する1行を追記
```

**既存プロジェクトでも安全です。** `dejavu init` はソースコードに一切触れません。`CLAUDE.md` と
`.gitignore` にも、まだ無い行だけを**追記**します。二度実行しても問題ありません。

**Xcode を使っている方へ**: `.dejavu/` はリポジトリのルート、つまり `.xcodeproj` の外に置かれるので、
ビルドに混ざることはありません。唯一の注意点は、Xcode 16 の **Synchronized Group（buildable folder）
の中に置かないこと**です。あの機能はフォルダ内のファイルを自動でターゲットに追加するためです。
リポジトリのルートに置く限り安全です。

あとはいつもどおり Claude Code を起動するだけです。

```bash
claude
```

## 知識のスコープ

```bash
dejavu init --global
dejavu init
```

この２つは何が違うのでしょうか？  
その違いが知識の**スコープ**です。  

あなたが使っているPCで常に憶えておきたいことはユーザーの記憶として憶えさせます（userスコープ）。あるプロジェクトだけで憶えておきたい事はプロジェクトの記憶として憶えさせます（projectスコープ）。

### 「これは全プロジェクトで憶えておいて」

user スコープは、**このリポジトリの話ではなく、あなた自身の話**を憶えさせる場所です。
コマンドもオプションも要りません。「どこでも」「いつも」「全プロジェクトで」といった言葉を添えるだけで、
Claude が勝手に user スコープを選んで記憶します。

```
あなた:  僕はいつも squash merge を使うから、どのプロジェクトでも憶えておいて

Claude: user スコープに保存しました。他のプロジェクトでも思い出せます。
```

user スコープに向いているのは、次のような内容です。

- **個人の好みや作業スタイル** — 「PR の説明は箇条書きで」「コミットメッセージは日本語で」
- **自分の環境の癖** — 「Python は Homebrew のものを使ってる」
- **プロジェクトをまたぐ文脈** — 別のプロジェクトに持ち越したい作業メモ

迷ったときの判断基準はひとつだけです。**「別のリポジトリでも同じことが言えるか?」**
言えるなら user スコープ、そのプロジェクト限定の話なら project スコープです。

- **project** — リポジトリ内の `.dejavu/` に置かれます。ほとんどの知識はこちら
- **user** — `~/.config/dejavu/` に置かれ、全プロジェクトから参照できます。個人メモや、
  プロジェクトをまたいだ文脈に使います

## dejavu のある一日

### 朝イチ ： 「昨日の続きを始めたい」

```
あなた:  昨日の続きをしたい

Claude:  昨日は CoreData のマイグレーションを完了し、PR #23 をマージしていますね。
         メモによると、次はロールバック経路のテストを書くところで、
         Sources/Store/Migration.swift:142 を見ればよいとあります。

         そこから始めますか?
```

Claude は**あなたが昨日残した引き継ぎメモ**を読みます。何を終えて、次に何をやるか、
どのファイルの何行目を見ればいいか。読み直しはありません。Claudeの決まり文句「まず状況を把握させてください」もありません。

### 日中 ： Claude が自分で使う

Claudeは自分でdejavuを使って記憶したり思い出したりしますので、あなたが管理する必要はありません。

- 見慣れないモジュールを読む**前に**、まず知識ベースを検索します。
- 前に調べたことがあれば、その続きから始めます
- 何かを理解したら、保存します
- 設計判断をしたら、決定事項や、**却下した案**を記録します。ここが普段いちばん失われやすい部分です

Claudeが処理実行の途中でときどき `dejavu` コマンドを叩くのが見えると思います。それが正常ですので「あぁ何か思い出してるんだな」と、放っておいてください。

### 脇道の作業：「後でやるから憶えておいて」

```
あなた:  その AppDelegate の警告、気になるけど今はいいよ。あとでやるから憶えといて。

Claude: 計画として保存しました。テストに戻ります。
```

**これが一番効く使い方かもしれません。** 作業中に何かが目に入る。手を止めて直すと本来の作業が中断する。
かといって放っておくと忘れる。「保存しといて」と言うだけで、作業を中断せずに取りこぼしを防げます。

あとで「保存してある計画を見せて」と言えば、Claude が一覧を出してくれるので、そこから選んで着手できます。

### 夕方 ： 「今日やったことを保存して」

```
あなた:  今日やったことを保存しておいて

Claude: 保存しました:「NEXT: ロールバック経路のテストを書く」
```

**明日の自分にメモを残すために、あなたが身につけるのは仕事終わりに「今日やったことを保存しておいて」という習慣だけです。** セッションの終わりの一言で、次回のセッションが爆速で開始できます。
この一言を忘れると `dejavu` はメモをとらず、翌日はゼロから始まってしまいます。

### いつでも ： 「最近何やってたっけ」

```
あなた:  今週何をやってたっけ?

Claude:  水曜日
           - CoreData マイグレーションをマージ (PR #23)
           - SwiftData を採用しない判断。理由も記録済み
         火曜日
           - ロールバック時のクラッシュを調査
```

日報や週報を作成する元ネタにできますし、数日休んで戻ってきて「どこまでやったんだっけ」となったときに効きます。

## 実際に言うのはこの4つだけ

ここまでの話は、結局この4つのフレーズに集約されます。コマンドを憶える必要はありません。

| こう言えば | Claude はこうする |
| --- | --- |
| **「昨日の続きをしたい」** | 最後の引き継ぎメモを読んで再開する |
| **「今日やったことを保存して」** | 次のセッション向けに引き継ぎメモを書く |
| **「それ、あとでやるから保存しといて」** | 作業を中断せずに脇道タスクを記録する |
| **「最近何をやってた?」** | 直近の作業を要約する |

## 何が、どこに保存されるのか

知識は6つのカテゴリのどれかに分類されます。Claude が適切なものを選ぶので、
あなたが意識する場面はほとんどありません。

| カテゴリ | 中身 | 例 |
| --- | --- | --- |
| `context` | セッションの引き継ぎメモ | 「完了: マイグレーションをマージ。次: ロールバックのテスト」 |
| `plan` | あとでやること | 「AppDelegate の警告を直す」 |
| `decision` | 設計判断と**却下した案** | 「SwiftUI ではなく UIKit を選んだ理由は…」 |
| `feature` | コードの仕組み | 「`Migration.swift` は3段階で動く…」 |
| `convention` | チームの規約 | 「view model は全て `ViewModel` で終わる」 |
| `note` | その他 | |

---

## 認証情報は保存されない

`dejavu` は、API キー・トークン・秘密鍵らしき文字列を含むテキストを拒否します。
Claude がうっかりシークレット入りのコード片を保存しようとしても、そこで止まります。

## チームで使う

`dejavu` は2種類の知識を使い分けています。

- **共有知識** — アーキテクチャ、設計判断、チーム規約。`.dejavu/*.md` に書き出され、
  **git で追跡**され、他のファイルと同じようにプルリクエストでレビューされます
- **ローカル知識** — Claude の調査キャッシュ。データベースの中だけに存在し、git 管理外です。
  使い捨てで、あなただけのものです

つまり、**チームメイトには決定と規約が届き、あなたの「この関数って何だっけ」的な調べ物は届きません。**
新しいメンバーがリポジトリをクローンすれば、その人の Claude は最初から
「なぜこのアーキテクチャなのか」を知っている状態になります。

データベース自体はコミットされません。Markdown からいつでも再構築できる索引にすぎないからです。

## 知識が古くなったら

コードは変わります。それについて書いたメモは変わりません。dejavu はこれを前提に設計されています。

各エントリは「最後に**実際のコードと照合した**のはいつか」を記録しています。それが古くなると
（調査キャッシュは7日、設計判断は30日。`.dejavu/config.toml` で変更可能）、検索結果に印がつきます。

```
⚠ [a3f01c8b] Migration は3段階で動く (feature) [STALE: 12 days since last check]
```

**古くなったエントリも、検索結果から消えるわけではありません。** 古い知識も手がかりとしては有用で、
消してしまうほうが害が大きいからです。Claude は「stale なエントリは現行コードと突き合わせてから使う」
よう指示されており、確認して正しければそのまま、ズレていれば修正するか、捨てて調べ直します。

記憶されている知識を棚卸ししたいときは「**古くなった知識を点検して**」と言えば、Claude が一つずつ現行コードと突き合わせて、直すもの・捨てるもの・そのままでいいものを整理してくれます。

## Claude Desktop と Cowork で使う

ここまでの話は「Claude がデータベースの隣でシェルコマンドを実行できる」ことが前提です。
ターミナルの Claude Code と Xcode 内蔵のエージェントはそうですが、**Claude Desktop と Cowork は違います。**
これらのシェルは隔離されたサンドボックスで動いていて、`~/.config/dejavu/` を見ることすらできません。

そこでは dejavu を MCP サーバーとして登録します。

```bash
dejavu install-mcp
```

アプリを再起動すれば、Claude が `search_knowledge` / `resume_knowledge` / `recent_knowledge` などを
ツールとして使えるようになり、**ターミナルと同じデータベース**を読み書きします。

**1点だけ違います: どのプロジェクトかを言う必要があります。** MCP サーバーはデスクトップアプリから
起動されるため、自分のカレントディレクトリを持ちません。どのリポジトリの話なのかを推測できないので、
名前を出してください。Claude がパスを渡してくれます。

```
あなた:  MyApp で昨日何をやってたっけ?
```

プロジェクトを言わなければ、**user スコープだけ**が使われます（個人メモとプロジェクトをまたぐ文脈）。
Claude Desktop から引きたいのは、たいていそちらでしょう。

## コマンド一覧

ほとんどのコマンドはClaudeが自動で実行するので、あなたがコマンドを打つ場面はほとんどありませんが、必要なときのために一覧しておきます。

| コマンド | 説明 |
| --- | --- |
| `dejavu init` | このプロジェクトをセットアップする |
| `dejavu init --global` | 全プロジェクト向けに指示書を設置する（最初の一度だけ） |
| `dejavu resume` | 最後の引き継ぎメモを表示する |
| `dejavu recent` | 直近の作業を日付ごとに表示する（既定は2日分） |
| `dejavu search "<検索語>"` | 知識ベースを検索する |
| `dejavu list --stale` | 見直しが必要なエントリを一覧する |
| `dejavu show <uid>` | エントリを全文表示する |
| `dejavu stats` | どれだけ溜まっているかを見る |
| `dejavu rm <uid>` | エントリを削除する |
| `dejavu install-mcp` | MCP サーバーを Claude Desktop / Cowork に登録する |

どのコマンドにも `--help` を付ければ全オプションが見られます。

## よくある質問

**保存を忘れないようにしないといけない?**  
Claude は指示書に従って作業しながら勝手に保存します。身につける価値があるのは、
作業を終える前に「今日やったことを保存して」と言う習慣だけです。
これがあって初めて、翌朝の「昨日の続き」が効きます。

**コードがどこかに送信される?**  
いいえ。dejavu はローカルのデータベースで、ネットワークアクセスは一切ありません。

**Claudeが遅くならない?**  
むしろ逆です。検索は数十ミリ秒で終わり、その代わりに Claude が数ファイル読むのを省けます。
毎ターン読み込まれる指示書も30行程度です。

**Claudeが間違ったことを保存したら?**  
「そのエントリは間違ってるから直して」と言えば直します。`dejavu rm <uid>` で削除もできます。
UID は検索結果の角括弧の中に出ている短いコードです。

**日本語で使える?**  
はい。全文検索では **「検索」「認証」のような2文字の日本語**（多くの検索エンジンが取りこぼす長さです）も確実にヒットするよう設計されています。

**Claude Code なしで使える?**  
使えます。普通の CLI なので `dejavu add` / `dejavu search` を手で叩けます。
ただしエージェントに使わせる前提で設計されているので、真価はそちらで発揮されます。

**プロジェクトから外すには?**  
`.dejavu/` を削除し、`CLAUDE.md` から追記された1行を消すだけです。他には何も触れていません。
